🚀 주린이 계산기 핵심 업데이트 내역
1. 하이브리드 통합 검색 엔진 탑재
하드코딩 제거: 과거 HTML 파일 안에 수동으로 적어두었던 20개의 한국 주식 목록(koreanStocks)을 완전히 폐기했습니다.

통합 검색: 이제 검색창에 입력하면 자체 구축한 **Render 파이썬 서버(한국 주식)**와 **FinnHub API(미국 주식)**를 동시에 찔러 결과를 가져옵니다.

디바운싱(Debouncing) 적용: 사용자가 타이핑할 때마다 서버를 공격하지 않도록, 입력이 끝나고 0.3초 뒤에만 검색을 실행하여 API 호출 제한(429 에러)을 완벽히 방어했습니다.

직관적 UI: 검색 결과 목록에 민트색 [KR], 파란색 [US] 뱃지를 달아 사용자가 국가별 주식을 쉽게 구분할 수 있게 되었습니다.

2. 홈 화면(index.html) 실시간 시세 전광판 개편
무거운 차트 제거: 로딩이 느리고 라이선스(권한) 에러가 잦았던 트레이딩뷰 위젯과 미니 차트를 과감히 걷어내고, 가볍고 정확한 **'숫자 중심의 데이터 표'**로 디자인을 개편했습니다.

수동 새로고침(🔄) 기능 도입: 자동으로 무한 새로고침을 돌려 서버를 혹사시키는 대신, 사용자가 가격 옆의 새로고침 버튼을 누를 때만 실시간 데이터를 갱신(On-Demand)하도록 아키텍처를 변경했습니다. 누를 때마다 1초간 빙글빙글 도는 애니메이션을 추가해 조작감을 높였습니다.

스마트 크롤링 연동: 한국 주식은 파이썬 서버가 네이버 금융을 크롤링하여 시가/고가/저가/전일종가와 함께 **"업데이트된 기준 시간"**까지 정확히 찍어줍니다.

UX 동선 개선: 홈 화면에서 한국 주식을 선택하면, 차트 대신 [배당 계산기에서 분석 →] 버튼이 크게 나타나 핵심 기능으로 자연스럽게 유도합니다.

3. 자바스크립트 안정성 및 무결점(Bulletproof) 처리
Null 에러 원천 차단: 화면이 다 그려지기도 전에 스크립트가 실행되어 발생하던 수많은 에러들을 document.addEventListener('DOMContentLoaded')라는 안전 지대 안으로 전부 통합하여 해결했습니다.

메모리 누수 방지: 새로운 주식을 검색할 때마다 이전 종목에 걸려있던 이벤트나 타이머를 깔끔하게 해제(clearInterval, destroy())하도록 설계했습니다.

4. 다크모드(Dark Mode) UI/UX 완치
디자인 붕괴 해결: 다크모드를 켰음에도 카드 배경이 하얗게 붕 뜨거나, 글씨가 배경색에 묻혀 안 보이던 버그를 CSS 변수(--card-bg, --text-primary 등)를 강제 적용하여 완치했습니다.

레이아웃 정렬: 라이트 모드에서 Y축 데이터(가격 등)가 삐뚤어지거나 우측 정렬이 깨지던 현상을 Flex 속성을 이용해 깔끔하게 교정했습니다.
